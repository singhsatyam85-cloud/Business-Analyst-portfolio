# Project Intake Note

## 1. Project Name
ABC School Web and Mobile Application

## 2. Client / Organisation Name
ABC School

## 3. Business Problem
ABC School wants to reduce manual work in attendance tracking and basic school data management. Currently, attendance and student / teacher records may be managed manually or in separate files.

## 4. Business Objective
Build a simple web and mobile application that allows teachers to mark student attendance and allows admin staff to manage student data, teacher data and attendance reports.

## 5. Proposed Users
- Admin Staff
- Teachers
- School Principal / School Owner

## 6. High-Level Scope
- Teacher login
- Teacher marks student attendance
- Teacher submits attendance
- Admin views attendance
- Admin manages student data
- Admin manages teacher data
- Admin views basic attendance reports
- Web application for Admin
- Mobile application for Teacher
- Role-based access for Admin and Teacher

## 7. Out of Scope
- Fee management
- Transport management
- Exam marks
- Timetable
- Parent login
- Payment gateway
- Notifications
- Complex analytics
- Third-party integration

## 8. Initial Assumptions
- ABC School will nominate one Product Owner from their side.
- Teachers will use the mobile application.
- Admin staff will use the web application.
- The first version will focus only on attendance and basic data management.
- No external system integration is required in Project 1.

## 9. Initial Risks / Clarifications Needed
- Need to confirm how ABC School currently manages attendance.
- Need to confirm number of students, teachers and classes.
- Need to confirm who will approve the requirements.
- Need to confirm whether the school wants Android only or both Android and iOS later.

## 10. Recommended Next Step
Arrange an initial discovery meeting with ABC School Owner, Principal, Admin Staff and Teacher Representative.
