# Backlog and Sprint Plan

## Project Name
ABC School Web and Mobile Application

## Delivery Approach
Agile Scrum with Sprint 0 discovery before development.

## Sprint 0: Discovery and Setup

| Item | Type | Owner | Output |
|---|---|---|---|
| Project intake review | Task | BA / Scrum Master | Project Intake Note |
| Stakeholder identification | Task | BA | Stakeholder Register |
| Discovery questions preparation | Task | BA | Discovery Questions |
| RACI preparation | Task | BA / Scrum Master | RACI Matrix |
| Initial scope confirmation | Task | Product Owner / BA | In-scope and out-of-scope agreement |

## Product Backlog

|---|---|---|---|---|---|---|

## Sprint 1 Goal
Deliver the core Teacher Attendance MVP so a teacher can log in, select class, mark attendance and submit it successfully.

## Sprint 2 Goal
Add admin visibility and basic student / teacher data management.

## Notes
MVP should stay focused on attendance and basic admin work. Fee management, transport, exam marks, parent login, payment gateway and notifications should remain out of scope for Project 1.
